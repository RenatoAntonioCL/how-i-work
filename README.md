# how-i-work

> It's not enough that it works. It has to be scalable, observable, and built to last.

This repo documents my engineering methodology — how I think about software, the standards every project I build follows, and where I'm going as a technical professional.

It exists for three reasons:
- So anyone can understand how I work before working with me.
- So every project I start from the same foundation, not from scratch.
- So my standards live somewhere permanent, not just in my head.
